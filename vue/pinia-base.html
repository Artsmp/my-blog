    pinia 基础

    artsmp 2022/1/18

    # 快速入门

    替代 vuex 的全局状态管理

    # 在项目中引入 pinia

    1. 安装：pnpm i pinia pinia-plugin-persist
    2. 创建 store 下的 index.ts 文件，并写入如下内容
    import { createPinia } from "pinia";
import piniaPluginPersist from "pinia-plugin-persist"; // pinia 持久化插件
const pinia = createPinia();
pinia.use(piniaPluginPersist);
export default pinia;
    1. 在 main.js 文件中使用
    import { createApp } from "vue";
import App from "./App.vue";
import pinia from "./store";
const app = createApp(App);
app.use(pinia);
app.mount("#app");

    pinia 中的数据结构是扁平化的

    # 定义和使用 store

    使用 defineStore 定义一个 store：

    import { defineStore } from "pinia";
import { getAllProducts, IProduct } from "../api/product";
import { CartProduct } from "./cart";

const useProductStore = defineStore("product", {
  // 相当于 vuex 中的 state
  // 这里的数据都是响应式的
  // 这里得使用箭头函数能获得完整的类型提示
  // state 返回的对象会自动进行类型推断
  state: () => {
    return {
      all: [] as IProduct[],
    };
  },
  getters: {},
  actions: {
    async getAllProduct() {
      const products = await getAllProducts();
      this.all = products;
    },
    decrementStock(product: IProduct) {
      const target = this.all.find((item) => item.id === product.id);
      if (target) {
        target.stock--;
      }
    },
    incrementStock(product: CartProduct, incCount: number) {
      const target = this.all.find(
        (item) => item.id === product.id
      ) as IProduct;
      target.stock += incCount;
    },
  },
});

export default useProductStore;

    # state

    • 默认情况下你可以直接读写 state
    • 你可以通过 $reset() 方法将 state 重置为初始值
    • vue2 使用 pinia (opens new window)
    • 还可以通过用 $patch(object|function) 方法修改 state
    • 你可以通过 store.$state = { counter: 6 } 设置一个新的 state，来整体替换旧的 state
    • 通过 $subscribe() 方法来观测 state 的变化。相比于 vue 的 watch 方法，订阅只会在 patch 时更新一次，例如使用 $patch 方法时。
    • 默认情况下，当组件被销毁时，绑定在组件上的 store 将被删除。如果想在组件卸载后保留他们，通过 $subscribe() 方法传递 { detached: true } 作为第二个参数，将会使当前 状态从组件中分离并保留。
    // 在组件中使用
import counterStore from "@/store/counter";
const store = useCounterStore();

// 直接读写 state
store.counter++;

// 重置为初始值
store.$reset();

// $patch
store.$patch({
  counter: 999,
});

store.$patch((state) => {
  state.counter++;
});

// 替换 state
store.$state = {
  name: "artsmp",
};

// 观测变化
store.$subscribe((mutation, state) => {
  mutation.type; // 'direct' | 'patch object' | 'patch function'
  mutation.storeId; // 'defineStore 时传入的第一个参数（仓库名）'
  mutation.payload; // 只有在 mutation.type = 'patch object' 时这个参数才是可用的

  // 我们可以在这进行本地存储
  localStorage.setItem(mutation.storeId, JSON.stringify(state));
});

    # getters

    getters 和 vue3 的 computed 完全等价。pinia 官方推荐使用箭头函数去定义某个 getters。但是 某个 getters 属性可能需要依赖于其他 getters，则这个时候我们就必须使用非箭头函数。这里有个坑，就是虽然定义 getters 时，我们可以使用传入的 state 参数访问状态值，也可以使用 this 访问，但是使用 this 时必须显式的定义返回值类型或者依然接收 state 但是不使用，再或者使用 JSDoc 去定义返回值。

    getters 定义时还可以返回一个函数，但是使用这个返回的函数时，不再具有缓存的功能。当然，我们可以利用闭包的机制，手动进行数据缓存。

    export const useCounterStore = defineStore("counter", {
  state: () => ({
    count: 0,
  }),
  getters: {
    doubleCount: (state) => state.count * 2,
    // 使用 state 会有自动类型推断
    addOne(state) {
      return state.count + 1;
    },
    // 需要显式的定义返回值类型
    minusOne(): number {
      return this.count - 1;
    },
    /**
     * 使用 JSDoc 也能自动获得返回值类型
     * @returns {number}
     */
    minusOne2() {
      return this.count - 1;
    },
    // 接收了 state 参数但未使用，也能自动获得返回值类型
    doubleMinusOne(state) {
      this.doubleCount - 1;
    },
  },
});

// 使用 getters
const store = useCounterStore();
console.log(store.doubleCount);

    # actions

    • 基本上和组件的 methods 方法等价，适合在这边处理一些 state 的数据逻辑。
    • actions 支持异步操作，其可以返回任何内容，返回的值得类型会被自动推断。
    • vue2 中使用 mapActions 将 store 的 actions 映射为组件中的方法。
    • store.$onAction(callback) 可以订阅这个 action，其接收一个回调函数。传递 true 作为第二个参数可以从当前组件分离订阅
    const unsubscribe = store.$onAction(
  ({
    name, // action 的名字
    store, // 当前这个 store 实例
    args, // action 接收的参数
    after, // action 返回的 promise 是成功态的时候执行这个钩子
    onError, // action 返回的 promise 是失败态的时候执行这个钩子
  }) => {
    const startTime = Date.now();
    after((result) => {
      console.log(`start: ${startTime}, end: ${Date.now()}`);
    });
  }
);

    # 在组件之外使用 store 实例

    有个顺序问题，就是必须等待 app 实例上存在 pinia 实例之后才能创建 store（useXxxStore）。所以如果我们想要在定义完创建 store 的函数后立即使用 store 实例，则需要用一个异步函数包裹一下，放到第二轮事件循环中去。

    在 store/xxxStore.ts 中：

    const useProductStore = defineStore('product', { ... })

setTimeout(() => {
    const productStore = useProductStore()
    productStore.$onAction(({ name, after, onError }) => {
        const startTime = Date.now()
        after((result) => {
            console.log(`name: ${name}, time: ${Date.now() - startTime}, result: ${result}`)
        })
        onError((reason) => {
        console.log(reason)
        })
    })
}, 0);

export default useProductStore
